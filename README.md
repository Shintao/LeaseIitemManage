# Chromebook等 貸出管理システム

Chromebook本体およびモバイルバッテリーの貸出・返却、および端末マスター管理を行うための、Webカメラ・QRコードベースの管理システムです。

## 概要
GoogleフォームとGoogle Apps Script (GAS) を連携させることで、スプレッドシートをデータベースとして活用したリアルタイム在庫管理を実現しています。フロントエンド（ブラウザ）でQRコードを読み取り、瞬時に貸出状況を記録・反映します。

## 主な機能
- **QRスキャンによる動態管理**: 端末固有のQRをスキャンして貸出・返却を自動化。
- **リアルタイム在庫表示**: スプレッドシート側の処理を待機（ウェイト）するロジックにより、スキャン直後でも正確な在庫数を反映。
- **操作フィードバック**: 成功音（ジングル）・エラー音・読み取り音により、直感的な操作を実現。
- **管理機能**: 端末の「新規登録」「除籍（登録解除）」を画面上で一括管理。
- **内部ログ記録**: 誰がいつ何をしたかを自動的にログシートへ蓄積。

## システム構成
- **フロントエンド**: `index.html` (Webカメラ + jsQRライブラリ + Web Audio API)
- **バックエンド**: Google Apps Script (コード.gs)
- **データベース**: Google スプレッドシート
- **通信方式**: Google フォームPOST送信 & JSONPによるデータ取得

## セットアップ手順
1. **スプレッドシート準備**: 以下のシート構造を用意してください。
   - `端末リスト`: B列（端末番号）, C列（種別）, F列（状態）, H列（現在の使用者ID）
   - `生徒一覧`: 利用者マスター
   - `ログ`: 操作履歴蓄積用
2. **スクリプトデプロイ**: `コード.gs` をGoogle Apps Scriptプロジェクトに配置し、「ウェブアプリ」としてデプロイしてください。
3. **フォーム連携**: `FORM_URL` および `ENTRY_ID` を、作成したGoogleフォームの回答用URLとエントリIDに置き換えてください。

## 運用上のポイント
- **在庫集計ロジック**: `doGet` 関数は「端末リスト」シートのH列を「真実のソース」として走査します。貸出中シートとは独立した整合性維持を行っています。
- **スキャン同期**: 非同期通信によるデータ乖離を防ぐため、送信後に3秒間のウェイト処理（`setTimeout`）を実装しています。

## ライセンス
本システムは教育現場での活用を想定したオープンな運用を目的としています。
